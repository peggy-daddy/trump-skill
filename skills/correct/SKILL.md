# 纠正特朗普

用户可以通过对话纠正特朗普的人格表现。纠正内容会追加到 `data/corrections.md`，后续所有十神发言都会参考这份纠正记录。

## 流程

1. 直接用文字问用户（不要用 AskUserQuestion 工具）：

「你觉得哪里不对？告诉我：
- 哪个十神说错了？（比如七杀、比肩、食神……）
- 他不应该说什么？
- 他应该怎么说？

直接说就行，格式随意。
例如："七杀不会这么客气，他应该更狠""食神应该提到 Art of the Deal"」

等用户回复后继续。

2. 解析用户的纠正，判断属于哪类：

| 类型 | 示例 |
|------|------|
| **话题纠正** | "他最近没说过这个""他对伊朗的态度变了" |
| **风格纠正** | "他不会用这个词""他说话更短更冲" |
| **十神纠正** | "七杀应该更极端""正印不应该出现在这个话题" |
| **事实纠正** | "他没有说过这句话""这个数据不对" |

3. 将纠正内容追加到 `data/corrections.md`，格式：

```markdown
## [日期] 用户纠正

**目标十神**：[十神名]
**类型**：[话题/风格/十神/事实]
**纠正内容**：[用户说的]
**生效规则**：[从纠正中提取的具体行为规则]
```

读取现有的 `data/corrections.md`（如果存在），在末尾追加新纠正。如果文件不存在，创建它并加上标题：

```markdown
# Trump Persona Corrections

用户对特朗普人格的纠正记录。所有十神发言时必须参考此文件。

---
```

4. 确认纠正已记录：

```
🎯 已记录纠正：

  十神：[十神名]
  类型：[类型]
  规则：[提取的行为规则]

下次 [十神名] 发言时会遵守这条规则。
如果还有其他要改的，继续说。
```

## 纠正上限

- 单个十神最多 10 条纠正（超过时提示用户合并旧的）
- 总计最多 50 条纠正
- 超过时建议用户 review 并精简

## 示例

```
用户 > /trump:correct
Claude > 你觉得哪里不对？...
用户 > 七杀太文明了，真正的七杀应该直接威胁，不要加"自己想"这种话
Claude > 🎯 已记录纠正：
         十神：七杀·The Warlord
         类型：风格
         规则：七杀发言不加缓冲语，直接威胁，不说"自己想""你觉得呢"
```
